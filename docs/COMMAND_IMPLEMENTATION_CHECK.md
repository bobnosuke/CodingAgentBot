## 2. 要件定義書との乖離・不足点

### A. 管理者コマンドの不足
- **`/config` の拡張**: 要件定義書 2449-2461 に記載されている「ユーザー同時利用数」「サーバー利用人数」の設定コマンドが未実装です。
- **DB永続化**: 現在の `/config` はメモリ上でのみ設定を保持しており、Bot再起動で消失します。`guild_settings` テーブルとの連携が必要です。

### B. `/setting` の詳細要件
- **利用状況の表示**: `/setting` 内の「利用状況確認」において、「本日の利用回数」「残り利用回数」の表示が「開発中」となっており、ロジックが組み込まれていません。

### C. Prefix Command の挙動統一
- 現在、私がハイブリッド化を行いましたが、一部のコマンド（特に Modal を使用するもの）において、Prefix Command から呼び出した際の UX が Slash Command と完全に同一ではない可能性があります（Discord の仕様上、Prefix メッセージから直接 Modal を開くことはできないため、ボタン等を経由させる必要があります）。

## 3. 今後の対応方針

1.  **`/config` の拡充 (Manus-iCloud 担当案)**:
    - `guild_settings` テーブルへの対応カラム追加。
    - `/config limit-user`, `/config limit-server` 等のサブコマンド追加。
2.  **利用制限ロジックの完成 (Manus-Beta 担当)**:
    - 1日50メッセージ制限、8秒Rate Limitの判定ロジック実装。
    - `/setting` の「利用状況確認」への反映。
3.  **モデルプリセットの完全連動 (Manus-Alpha 担当)**:
    - `/setting` で選んだモデルが実際の AI 処理に反映されるように修正。
