# CodingAgent Multi AI Autonomous Agent System 改修計画書

## 1. 目的

現在のCodingAgentは単一AIモデルによるコード生成システムですが、これをユーザーがAIモデル名を選択せず、品質レベル（高速、標準、高品質）のみを選択する構成に変更します。内部では品質に応じたAIモデルを自動選択し、AIフォールバック、Dockerセッション管理、Agent Toolシステムなどを実装することで、より堅牢で使いやすい自律型エージェントシステムを目指します。

## 2. 全体改修概要

### 2.1. ユーザーインターフェースの変更

*   AIモデル選択UIを削除し、品質選択UI（高速、標準、高品質）を追加します。
*   設定画面に現在利用中のAI、Cooldown中のAI、今日の利用回数、残り利用可能数などの表示項目を追加します。

### 2.2. AIモデル管理システムの導入

*   AIモデル名をコードに直接記述せず、`config/models.json` または `config/ai_models.py` に外部化します。
*   指示書に記載された各カテゴリ（Coding系高品質候補、Reasoning/Architecture系、Standard Coding系、Fast系）のモデルを登録します。

### 2.3. 品質レベルシステムの導入

*   ユーザーが選択した品質レベル（高速、標準、高品質）に応じて、適切なAIモデル群を自動選択するロジックを実装します。
*   「高品質」選択時にはCoding高品質モデルとReasoning系モデルを組み合わせることを検討します。

### 2.4. AIフォールバックとレートリミット管理

*   AI API実行時に429/503エラーやレートリミットが発生した場合、自動的にバックアップAIへ切り替えるフォールバック機構を実装します。
*   モデルごとに`model_status`（cooldown、retry_afterなど）、成功時間、エラー回数、使用量を管理するシステムを導入します。
*   フォールバックが全て失敗した場合、Discord上でユーザーに状況を通知し、再実行オプションを提供します。

### 2.5. Agent Toolシステムの導入

*   AIが直接ファイル操作を行わないよう、以下のAgent Toolを実装します。
    *   `list_files()`
    *   `read_file()`
    *   `create_file()`
    *   `edit_file()`
    *   `delete_file()`
    *   `move_file()`
    *   `execute_project()`
    *   `check_project_status()`

### 2.6. Docker検証システムとImage管理の改修

*   AIが生成したコードをセッション専用のDockerイメージで検証するフローを必須とします。
*   Dockerイメージ名を`coderagent-exec:{session_id}`形式に変更し、ユーザーごとに異なるrequirements.txtやパッケージに対応できるようにします。
*   コード実行失敗時にはエラーログをAIにフィードバックし、最大3回まで修正・再実行を試みます。

### 2.7. Project Backup管理の導入

*   プロジェクトファイルを`project/main/`、`project/backup/`、`project/logs/`、`metadata.json`の構造で管理します。
*   初回保存時には`main`と`backup`の両方に保存し、2回目以降は`backup`に直前の`main`の内容を、`main`に最新のコードを保存するロジックを実装します。

### 2.8. 将来対応への考慮

*   有料プラン、ユーザーAPI Key登録、モデル直接指定、カスタムモデルなどが将来的に追加可能な設計とします。

## 3. 影響を受ける主要ファイルと変更点

### 3.1. AI関連

*   **`modules/ai/openrouter.py`**: OpenRouter API呼び出し箇所、モデル指定箇所、retry/timeout/API error処理の改修。フォールバックロジックの追加。
*   **`modules/ai/agent.py`**: `execute_task()`におけるAIへのプロンプト、self-correction loop、tool利用の改修。Docker検証システムとの連携強化。
*   **`modules/ai/views.py`**: Discord UI関連の変更に伴う修正。
*   **`modules/ai/prompts_cerebras.py`**: プロンプト内容の調整。

### 3.2. Executor関連

*   **`modules/executor/docker_executor.py`**: Docker image生成、container実行、requirements処理、session_id管理、error feedbackの改修。特にイメージ名の変更（`coderagent-exec:{session_id}`）。

### 3.3. Agent Tool関連

*   **`modules/file/manager.py`**: `list_files()`, `read_file()`, `create_file()`, `edit_file()`, `delete_file()`, `move_file()`の実装または既存機能の拡張。
*   **新規ファイル**: `modules/agent_tools/` ディレクトリなどを検討し、Agent Toolのロジックをカプセル化。

### 3.4. Config関連

*   **`config.py`**: AIモデル設定の外部化、品質設定の追加、`DEFAULT_AI_MODEL`の変更。
*   **`cogs/setting.py`**: Discord設定画面のUI変更（AIモデル選択削除、品質選択追加、ステータス表示追加）。
*   **`docker-compose.yml`**: Dockerイメージ名の変更に対応するための調整。
*   **`.env.example`**: 新しい設定項目（例: AIモデル管理ファイルのパス）の追加。
*   **新規ファイル**: `config/models.json` または `config/ai_models.py` を作成し、AIモデル情報を一元管理。

### 3.5. Discord UI関連

*   **`cogs/` ディレクトリ内の関連ファイル**: AIモデル選択UIの削除、品質選択UIの追加、設定画面の表示項目追加。
*   **`modules/ai/views.py`**: UIコンポーネントの修正。

### 3.6. Database Model関連

*   **`modules/database/models.py`**: `User`モデルに品質レベル設定、`Session`モデルに現在のAIモデル、`UsageLog`に詳細な利用状況（エラー回数、cooldown情報など）を格納するためのフィールド追加を検討。また、`model_status`を管理するための新規テーブルの追加も検討。
*   **`modules/database/repository.py`**: 新しいモデルやフィールドに対応するためのCRUD操作の追加・修正。
*   **`migrate_db.py`**: データベーススキーマ変更に伴うマイグレーションスクリプトの更新。

### 3.7. Project Backup関連

*   **`modules/project/manager.py`**: プロジェクトの保存構造を`main/`, `backup/`, `logs/`, `metadata.json`に対応させるように改修。
*   **`modules/session/manager.py`**: セッション終了時のプロジェクトクリーンアップロジックの調整。

### 3.8. その他

*   **`requirements.txt`**: 必要なライブラリの追加・更新。
*   **`tests/` ディレクトリ内の関連ファイル**: 各機能改修に伴うテストコードの追加・修正。

## 4. 実装フェーズ

1.  **AIモデル管理システムの実装**: `config/models.json`の作成と、AIモデル選択ロジックの変更。
2.  **品質レベルシステムの実装**: ユーザーの品質選択に応じたAIモデルの動的選択ロジックの実装。
3.  **AIフォールバックとレートリミット管理の実装**: エラーハンドリング、バックアップAIへの切り替え、モデルステータス管理、Discord通知の実装。
4.  **Agent Toolシステムの実装**: ファイル操作ツール（`list_files`など）とプロジェクト実行ツールの実装。
5.  **Docker検証システムとImage管理の改修**: `docker_executor.py`の修正、セッションIDに基づくイメージ名の変更。
6.  **Project Backup管理の実装**: プロジェクト保存構造の変更とバックアップロジックの実装。
7.  **Discord UIの改修**: `cogs/setting.py`および関連ファイルの修正、品質選択UIと設定画面の表示項目追加。
8.  **データベーススキーマの更新**: `models.py`と`repository.py`の修正、マイグレーションスクリプトの作成。
9.  **テストとデバッグ**: 全体の統合テストとデバッグ。
10. **ドキュメント更新**: `README.md`やその他の関連ドキュメントの更新。

この計画に基づき、各フェーズで詳細な実装を進めていきます。
