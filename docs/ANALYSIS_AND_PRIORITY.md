## 2. ギャップ分析

### A. `/setting` コマンドとモデルプリセット (優先度: 高)
- **現状**: `cogs/setting.py` は存在するが、`modules/database/repository.py` に対応するメソッドが不足しており、実際に DB へ保存・反映される部分が未完成。
- **要件**: ユーザーが「高品質」「バランス」「節約」を選択し、それが `/coding start` 時の AI モデル選択に反映される必要がある。

### B. 利用制限 (Rate Limit / Usage Limit) (優先度: 高)
- **現状**: `models.py` に `UsageLog` はあるが、制限をチェックするロジック (`modules/security/limits.py` 等) が存在しない。
- **要件**: 1日50メッセージ制限、8秒に1回の Rate Limit。

### C. サーバー設定の永続化 (優先度: 中)
- **現状**: `/config set-category` はあるが、メモリ上のみで DB に保存されていない。
- **要件**: サーバーごとにコーディングルームを作成するカテゴリを永続化する。
