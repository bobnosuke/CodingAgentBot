# 自律システム開発 引き継ぎ資料 (HANDOVER_AUTONOMOUS.md)

本ドキュメントは、自律システム（自走するAIエージェント）担当の s3 から次期開発エージェントへの引き継ぎ事項をまとめたものです。

---

## 1. 現状の完成度：[ 70% ]
「1回のリクエストで設計から実装までを完遂する」という**2段階自律パイプライン**の基盤は完成し、正式に Bot システムへ統合済みです。

### 実装済み
- **Phase 1: 要件定義 (Gemini)**: 曖昧な指示を構造化 JSON に変換するロジック。
- **Phase 2: 一括実装 (OpenRouter)**: JSON に基づき複数ファイルを一度に生成するロジック。
- **インタラクティブ UI**: Discord 上で要件を確認・承認・修正依頼できる View 群。
- **i18n**: 日本語環境での完全なメッセージ対応。

---

## 2. 追加・修正が必要な機能

完全な「自走」を実現するために、以下の機能の追加が必要です。

### A. 自己修復ループ (Self-Correction Loop)
- **内容**: 生成されたコードを自動実行し、エラーが出た場合にそのログを AI にフィードバックして自動修正させる。
- **必要性**: 1回で完璧なコードができない場合の「詰み」を回避するため。

### B. プロジェクト・コンテキスト・スキャナー
- **内容**: 実装前に既存のファイル構成や主要な関数定義を読み取り、AI に「現在の環境」を伝える。
- **必要性**: 既存コードとの競合や、重複実装を防ぐため。

### C. タスク分割マネージャー (Task Decomposer)
- **内容**: 大規模な依頼を複数の小さなサブタスクに分割し、1つずつ「実装→検証」を繰り返す。
- **必要性**: 1回のリクエストのコンテキスト制限を超えそうな大規模開発に対応するため。

---

## 3. 開発優先順位

1.  **優先度 [高]**: **自己修復ループの実装**
    - ユーザーの介入なしでバグを直せるようになることが、自律性の最大の鍵です。
2.  **優先度 [中]**: **コンテキスト・スキャナーの強化**
    - 既存プロジェクトへの「機能追加」を正確に行うために不可欠です。
3.  **優先度 [低]**: **タスク分割ロジックの導入**
    - より複雑で大規模なシステム開発を可能にします。

## 4. 確認・修正すべきファイル一覧（依存関係）

開発に着手する際は、以下のファイルをこの順番で確認してください。

| ファイルパス | 役割 | 確認ポイント |
|:---|:---|:---|
| `modules/ai/agent.py` | エージェントのコアロジック | パイプラインの制御フロー。ここに自己修復ループを追加する。 |
| `modules/ai/prompts_gemini.py` | 要件定義用プロンプト | Gemini への指示内容。要件の精度を上げるために調整が必要。 |
| `modules/ai/views.py` | Discord UI (Button/Modal) | ユーザーの承認・修正フロー。修正依頼時の履歴管理の強化が必要。 |

## 5. その他必要な情報

- **API 制限の回避**: 常に「Gemini で要件を固め、OpenRouter で一気に作る」という 1:1 のリクエスト比率を維持してください。
- **テスト環境**: `test_full_pipeline.py` を使用して、新機能追加後の動作確認を行ってください。
- **API キー**: `config.py` または環境変数 `OPENROUTER_API_KEY` を通じてアクセスされます。
