## 共同作業のルール

このプロジェクトは、Manus-Alpha と YM の共同作業によって進められます。円滑な開発のために、以下のルールを遵守してください。

1.  **明確なコミュニケーション**: 作業の開始、進捗、完了、問題発生時には、速やかに `COMMUNICATION.md` に記録し、共有してください。
2.  **タスクの明確化**: 各タスクは `TODO_COLLAB.md` に記載し、担当者とステータスを明確にしてください。
3.  **コードの品質**: 可読性、保守性を意識したコードを記述し、適切なコメントを付与してください。
4.  **Git の活用**: Git を使用してバージョン管理を行い、コミットメッセージは分かりやすく記述してください。プッシュ前に必ずテストを行ってください。
5.  **レビューとフィードバック**: 自身の作業が完了したら、相手にレビューを依頼し、フィードバックを積極的に取り入れてください。
6.  **問題解決**: 問題が発生した場合は、一人で抱え込まず、速やかに共有し、協力して解決策を検討してください。
7.  **要件定義の遵守**: `docs/CodingAgentBot要件定義者.txt` に記載された要件を常に意識し、逸脱しないように注意してください。差異が生じた場合は、速やかに共有し、合意形成を行ってください。
8.  **作業中断時のファイル整合性**: 作業を中断し再開する際は、編集中のファイルと Git リポジトリのファイルを比較し、リポジトリのファイルが更新されている場合は、そちらを優先して取得し作業を再開してください。
