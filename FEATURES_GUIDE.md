# CoderAgent 機能ガイドブック

本ドキュメントは、CoderAgent Bot に現在実装されている全機能、および今後の実装予定をまとめたものです。開発者や管理者が現在の進捗状況を確認するためのリファレンスとして活用してください。

---

## 1. ユーザー向けコマンド (スラッシュコマンド)

ユーザーが Discord 上で直接実行できる主要なコマンドです。

### ⚙️ 設定関連 (`/setting`)
ユーザーごとの個別設定を行うためのコマンドです。実行すると、言語設定に基づいた公開パネルが表示されます。

| 機能 | 内容 |
| :--- | :--- |
| **APIキー管理** | OpenRouter APIキーの登録・削除。キーは AES-256 で暗号化保存されます。 |
| **モデル選択** | AIの性能プリセット（高品質・バランス・節約）の切り替え。 |
| **言語設定** | Botの応答言語（日本語・英語）の切り替え。 |
| **利用状況確認** | 現在のモデル、登録済みキーの有無、1日のメッセージ使用量を確認。 |

### 🚀 コーディング関連 (`/coding`)
AIコーディングセッションを開始・管理するためのコマンドです。

| コマンド | 機能 |
| :--- | :--- |
| **`/coding panel`** | コーディング開始やプロジェクト一覧を表示するための総合パネル。 |
| **`/coding start`** | 新しいコーディングセッションを開始し、専用の **CodingRoom** を作成。 |
| **`/coding end`** | 現在のセッションを終了し、CodingRoom を削除（確認ダイアログ付き）。 |

---

## 2. CodingRoom 内機能 (プレフィックスコマンド)

`/coding start` で作成された専用チャンネル内でのみ利用可能なコマンドです。

| コマンド | 機能 |
| :--- | :--- |
| **`!download`** | セッション内で生成されたファイルを個別に、または ZIP 形式で一括ダウンロード。 |
| **`!close`** | セッションを終了（`/coding end` と同等）。 |
| **AI自動応答** | チャンネル内での発言に対し、AIが自動的にコード生成や回答を行います。 |

---

## 3. 管理者・オーナー向け機能

特定の権限（Admin または Bot Owner）を持つユーザーのみが実行可能です。

### 🛠️ オペレーターコマンド (`/operator`)
| コマンド | 権限 | 機能 |
| :--- | :--- | :--- |
| **`/operator health`** | Admin | Botのレイテンシとデータベース接続状態の確認。 |
| **`/operator stats`** | Admin | 総ユーザー数、総セッション数、AI呼び出し回数の統計表示。 |
| **`/operator shutdown`** | Owner | Botの安全なシャットダウン。 |

### 🔄 メンテナンス
| コマンド | 権限 | 機能 |
| :--- | :--- | :--- |
| **`!reload`** | Owner | Botを再起動せずに Cog（プログラムモジュール）を最新の状態に更新。 |

## 4. システム基盤

| カテゴリ | 実装内容 |
| :--- | :--- |
| **多言語対応 (i18n)** | 日本語と英語に対応。`locales/*.json` でメッセージを一元管理。 |
| **セキュリティ** | APIキーの AES-256 暗号化保存、チャンネル権限の自動制御。 |
| **データ永続化** | SQLite (aiosqlite) を使用。ユーザー情報、セッション、会話履歴を保存。 |
| **AI連携** | OpenRouter API を介した多種多様な LLM の利用。 |

## 5. 今後の実装予定 (ロードマップ)

コードレビューおよび要件定義に基づき、今後追加・改善が予定されている機能です。

### 短期的な改善
- [ ] **CodingRoom コマンドの完備**: `!list` (ファイル一覧), `!get` (ファイル取得), `!readme` (README表示) の実装。
- [ ] **パネル機能の有効化**: `/coding panel` からのプロジェクト一覧表示、詳細確認、名前変更の実装。
- [ ] **APIキー更新ロジック**: 既存キーがある場合にスムーズに上書きできる仕組みの改善。

### 中長期的な機能追加
- [ ] **自動キャッシュ削除**: 3日間更新がないセッションや一時ファイルの自動クリーンアップ。
- [ ] **利用制限の強化**: サーバーごとの最大同時セッション数や、詳細なレートリミットの設定。
- [ ] **PostgreSQL への移行**: 大規模運用を見据えたデータベースのアップグレード。
- [ ] **詳細な統計ログ**: `UsageLogRepository` を活用した、より詳細な利用傾向分析機能。
